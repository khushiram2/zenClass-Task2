Differences between HTTP/1.1 and HTTP/2

HTTP/1.1:

  • 1. HTTP/1.1 is slower than HTTP/2 because HTTP/1.1 loads data one after another. On the other hand, HTTP/2 uses one TCP connection and divides the data into multiple streams. This allows simultaneous loading of data, avoiding blocking issues.
  • 2. Both HTTP/1.1 and HTTP/2 compress data in HTTP packets to speed up data transfer. However, HTTP/2 uses a more advanced compression called HPACK, which results in smaller header packets compared to HTTP/1.1. This reduces the overall data size and makes HTTP/2 faster.
  • 3. HTTP/2 offers weighted prioritization, allowing developers to prioritize requests for resources. In HTTP/1.1, requests were handled in the order received, but HTTP/2's prioritization feature allows for more efficient resource allocation.

Objects and their representation in JavaScript:

  • 1. Objects come under composite data types in JavaScript. They can store multiple values in key-value pairs, enclosed in curly braces ({})
  • 2. Object representation in JavaScript is done by defining variables and assigning key-value pairs to them:
    • var obj = {
  Name: "xyz",
  releaseYear: "20**",
  geners: ["romance", "action", "sci_fi"]
};
  • 3. To add a new key-value pair to an object, you can simply assign a value to the corresponding key:
    • obj.ranking = 4;
  • 4. To update the value of an existing key, you can assign a new value to it:
    • obj.ranking = 14;
  • 5. To delete a key-value pair from an object, you can use the `delete` keyword:
    • delete obj.releaseYear;
